Spatial Import Tool

空间数据导入工具 - 代码逻辑架构分析

Tauri 2 React TypeScript Rust GDAL PostgreSQL/PostGIS

⚛️ 前端架构 (React + TypeScript)

App.tsx - 主应用组件

应用的入口组件，管理三个步骤状态：files → import → progress。协调所有子组件的交互。

useImport Hook - 状态管理核心

自定义 Hook，集中管理所有应用状态，包括文件选择、数据库配置、导入设置、进度追踪等。通过 Tauri invoke 与后端通信。

FileSelector.tsx - 文件选择器

负责文件选择、图层浏览、显示文件信息（格式、要素数量、几何类型、字段列表）。

DbConfig.tsx / DbConfigList.tsx - 数据库配置

管理数据库连接配置（PostgreSQL/Dameng），支持保存、加载、测试连接。

ImportConfig.tsx - 导入配置

配置表名、空间参考系统(SRS)、导入模式（CreateNew/Append/Replace）。

Progress.tsx - 进度显示

显示导入进度条、已导入数量、错误信息、完成状态。

🦀 后端架构 (Rust/Tauri)

lib.rs - 应用入口

初始化 GDAL 环境、注册 Tauri 命令、配置插件（opener、dialog、fs）。

file_commands.rs - 文件操作命令

list_files: 获取多个文件信息
get_file_info: 获取单个文件详情
list_layers: 列出文件中的所有图层
get_supported_drivers: 返回支持的格式

db_commands.rs - 数据库命令

test_connection: 测试数据库连接
save_config: 保存配置到本地
load_config: 加载配置列表
delete_config: 删除配置

import_commands.rs - 导入核心逻辑

import_file: 异步执行导入，通过事件发送进度
read_feature_batch: 批量读取要素
batch_insert: 使用 COPY 命令批量插入

postgres.rs - 数据库层

PostgreSQL 连接管理、配置持久化到 ~/.config/spatial-import-tool/db_config.json

gdal/mod.rs - GDAL 处理器

使用 GDAL 库读取 Shapefile、GeoPackage、GeoJSON、KML 格式，获取图层、字段、空间参考信息。

🔄 数据导入流程

1

用户选择文件

  • 前端调用 Tauri dialog 插件打开文件选择对话框
  • 调用 list_files 命令获取文件信息
  • GDAL 读取文件元数据（格式、图层、字段、SRS）
↓
2

配置数据库连接

  • 用户填写数据库连接参数或选择已保存的配置
  • 调用 test_connection 验证连接
  • 成功后自动保存配置到本地 JSON 文件
↓
3

设置导入参数

  • 设置目标表名（自动从文件名生成）
  • 选择空间参考系统（默认 EPSG:4326 或从文件读取）
  • 选择导入模式：CreateNew / Append / Replace
↓
4

执行导入

  • 调用 import_file 启动后台线程
  • 创建 PostgreSQL 连接池（r2d2）
  • 根据模式创建/删除/检查表结构
  • 添加 geom 字段（GEOMETRY 类型）和空间索引
↓
5

批量数据处理

  • 循环读取要素批次（动态批次大小：100-1000）
  • GDAL 将几何转换为 WKT 格式
  • 使用 PostgreSQL COPY 命令批量插入
  • 发送 import-progress 事件更新进度
↓
6

完成导入

  • 发送 import-complete 事件
  • 前端显示导入结果（成功数量、错误、耗时）
  • 用户可重置回到文件选择步骤

命令 参数 返回值 功能描述

📋 导入模式详解

CreateNew - 创建新表

创建新表，如果表已存在则失败。表结构包括：
• gid SERIAL PRIMARY KEY
• geom GEOMETRY
• 各字段（根据 GDAL 类型映射）
• 空间索引 (GIST)

Append - 追加数据

追加数据到现有表。前提条件：
• 表必须已存在
• 必须有 geom 字段
• 字段结构应匹配（不匹配的字段会被忽略）

Replace - 替换表

删除现有表并重新创建。相当于：
• DROP TABLE IF EXISTS
• 然后执行 CreateNew 流程

💡 关键技术实现

异步导入机制

import_file 在后台线程执行，立即返回临时结果。
通过 Tauri 事件系统发送进度更新：
• import-progress: 进度更新
• import-complete: 导入完成

批量插入优化

使用 PostgreSQL COPY 命令批量插入：
• COPY table FROM STDIN
• 动态批次大小（100-1000条）
• 连接池管理（r2d2）

GDAL 类型映射

GDAL 字段类型 → PostgreSQL 类型：
• Integer → INTEGER
• Integer64 → BIGINT
• Real → DOUBLE PRECISION
• String → VARCHAR(255)
• DateTime → TIMESTAMP

配置持久化

数据库配置存储位置：
~/.config/spatial-import-tool/db_config.json

支持多个命名配置，记录活动配置。
